# Track Specification: Repository Scrub and Audit

**Track ID:** repo_scrub_audit_20260308
**Type:** Chore / Refactor
**Goal:** Cleanse the remote repository of non-essential development artifacts and AI markers while performing a high-rigor audit for performance and scientific improvements.

## 1. Overview
This track prepares the repository for public/reviewer visibility. It involves moving all "behind-the-scenes" context (Conductor tracks, decision logs, manuscript drafts) to a local-only environment and scrubbing the remote-facing code of any evidence of AI agent usage. Simultaneously, a comprehensive audit will be performed to identify optimization and rigor-alignment opportunities.

## 2. Functional Requirements

### 2.1 Artifact Relocation & Git Management
- Create a `local_only/` directory to house all non-public artifacts.
- Move the following to `local_only/`:
    - `conductor/` (tracks, templates, workflow)
    - `context/` (decision logs, assumptions registry, jurisdiction profiles) - *Exception: Essential configs needed for CI/CD.*
    - `docs/*.md` (manuscript drafts, methods sections, internal reports)
    - All cache files, AI logs, and temporary artifacts.
- Update `.gitignore` to explicitly ignore the `local_only/` folder and ensure these paths are removed from the git index.
- **Path Robustness:** All scripts must be refactored to use environment-aware paths (e.g., checking for data in `context/` then falling back to `local_only/context/`).

### 2.2 Strict AI Evidence & PII Scrubbing
- Scan all source files (`src/`, `scripts/`, `tests/`) for AI-generated headers, comments, or metadata.
- **PII Scrub:** Remove absolute local paths (e.g., `C:\Users\...`).
- Remove agent-specific markers (e.g., "Generated by Gemini", "Refinement round N").
- Standardize code comments to maintain a professional, human-authored appearance.
- **Dead Code Elimination:** Identify and remove unused functions or imports.

### 2.3 Comprehensive Repository Audit
- Review `https://github.com/edithatogo/ginsim` for:
    - **Performance/Optimization:** Identify JAX/XLA bottlenecks and memory efficiency opportunities.
    - **Scientific Rigor:** Assess alignment with CHEERS 2022 and GRADE standards.
    - **Documentation/UX:** Improve reviewer-facing READMEs and dashboard navigation.
    - **Citation:** Ensure `CITATION.cff` is present and accurate.

## 3. Non-Functional Requirements
- **Zero-Downtime Dashboard:** The Streamlit app must deploy and run without errors after the scrub.
- **Local Continuity:** The developer must be able to continue using Conductor tools locally from within the `local_only/` folder.
- **Clean-Room Proof:** The repository must pass all tests when cloned into a completely fresh environment without the `local_only/` folder.

## 4. Acceptance Criteria
- [ ] `local_only/` folder is established and contains all relocated artifacts.
- [ ] `.gitignore` is updated; `git status` shows no "leaked" context files.
- [ ] Source code is cleansed of all identifiable AI markers and PII.
- [ ] `CITATION.cff` is created.
- [ ] A "Repository Improvement Report" is generated in `local_only/docs/IMPROVEMENT_REPORT.md`.
- [ ] The model and dashboard pass all local smoke tests in a clean environment.

## 5. Out of Scope
- Major architectural rewrites (unless identified as critical during the audit).
